Proposal for a Verified Dating Platform
Prepared for: The Curious Bunny Studios
Prepared by: Sumedhav, Sparsh (Dev Team)
Application Type: Cross-Platform Mobile Application
1. Executive Summary
Online dating platforms have significantly changed how people connect globally.
However, modern dating applications face serious challenges such as catfishing,
identity fraud, privacy concerns, and lack of trust between users.
This proposal outlines the development of a cross-platform dating application
focused on identity verification, user safety, and privacy-first communication.
The primary goal of the platform is to create a trust-driven dating ecosystem
where users can evaluate the credibility of profiles through tiered verification
levels, while maintaining strong privacy controls and secure communication.
The platform will be built using React Native for cross-platform mobile
development and a Node.js backend architecture, ensuring scalability and
performance for a global audience.

2. Problem Statement
Modern dating platforms face several persistent issues that reduce user trust
and safety.
2.1 Catfishing & Fake Profiles
Many users create fake profiles using stolen images or misleading information to
deceive others.
This results in:
● Emotional harm
● Scams and fraud
● Reduced trust in the platform
2.2 Privacy Risks
Users are often exposed to privacy risks such as:
● Screenshots of private conversations
● Unwanted sharing of personal information
● Early exposure of sensitive data like phone numbers or addresses
2.3 Inefficient Matching Systems
Many applications rely on simplistic matching algorithms that do not consider
important user boundaries.
Users often want:
● Non-negotiable preferences
● Lifestyle compatibility
● Cultural or religious considerations
Without proper filtering, user experience deteriorates.

3. Proposed Solution
We propose a Verified Dating Platform designed to prioritize trust, privacy, and
compatibility.
The platform introduces the following key innovations:
1. Incentivized Verification System
Users can choose multiple levels of verification to increase their trust score.
2. Privacy-First Communication
Strict privacy controls prevent unauthorized sharing of sensitive information.
3. Intelligent Matching Filters
A hybrid filtering system using Hard Filters and Soft Filters ensures
compatibility while maintaining inclusivity.
4. Core Features
4.1 Multi-Level Profile Verification
To eliminate catfishing, the platform introduces a three-tier verification model.
Level 1 — Basic Verification
● Face recognition during onboarding
● Live selfie validation
● Ensures the user matches uploaded photos
Level 2 — Trustworthy Verification (Optional)
Users can connect verified social accounts such as:
● Instagram
● Facebook
● Snapchat

Additionally:
● Government ID verification may be performed.
Level 3 — Secure Verification (Optional Future Feature)
A deeper trust level may include:
● Background verification
● Criminal record checks (subject to legal and regional compliance)
This tier is optional to maintain low onboarding friction while enabling higher
trust levels for serious users.
4.2 Privacy Protection System
User privacy will be protected through multiple mechanisms.
Screenshot Protection
● Android: Screenshot blocking using secure window flags
● iOS: Screenshot detection with warning and potential content masking
Personal Data Protection
Sensitive information such as:
● Phone numbers
● Addresses
will remain hidden until both users mutually approve sharing after matching.
Reporting System
Users can report profiles for:
● harassment
● inappropriate content
● suspicious behaviour

4.3 Matching System
The application will implement a hybrid matching model.
Hard Filters (Non-Negotiable)
These filters strictly control which profiles are shown.
Examples include:
● Orientation
● Religion
● Relationship goals
● Age range
Soft Filters
These influence but do not strictly limit matching.
Examples include:
● Distance radius
● Lifestyle preferences
● Dating intentions (casual / serious / unsure)
This approach ensures inclusivity while respecting user boundaries.

6. Location Based Matching
The application will support radius-based matching.
Users can define a preferred distance range (for example 5km–100km).
Location will be used only to:
● show nearby profiles
● improve matching relevance
Exact locations will never be shared publicly.
7. Safety Features
